Fundamentals of Computer & Computing Concepts

Q.1 Explain the basic components of a computer system with a neat diagram. 4 / 6 Marks ▼
This is a core question — draw the block diagram and explain each component clearly.

A Computer System is an electronic device that accepts data as input, processes it, and produces meaningful results as output. It consists of the following major components:

Block Diagram

┌─────────────┐ ┌──────────────────────────────────┐ ┌──────────────┐ │ INPUT │────▶│ CENTRAL PROCESSING UNIT │────▶│ OUTPUT │ │ DEVICES │ │ ┌─────────────┐ ┌────────────┐ │ │ DEVICES │ │ Keyboard │ │ │ ALU │ │ Control │ │ │ Monitor │ │ Mouse │ │ │(Arithmetic │ │ Unit (CU) │ │ │ Printer │ │ Scanner │ │ │Logic Unit) │ │ │ │ │ Speaker │ └─────────────┘ │ └─────────────┘ └────────────┘ │ └──────────────┘ └──────────────┬───────────────────┘ │ ┌──────▼──────┐ │ MEMORY │ │ (Primary) │ │ RAM / ROM │ └─────────────┘ │ ┌─────────────▼─────────────┐ │ SECONDARY STORAGE │ │ Hard Disk, CD, Pen Drive │ └────────────────────────────┘

1. Input Unit

Accepts data and instructions from the user and converts them into binary form understood by the computer. Examples: Keyboard, Mouse, Scanner, Microphone.

2. Central Processing Unit (CPU)

It is the brain of the computer. It has two sub-units:

  • ALU (Arithmetic Logic Unit): Performs all arithmetic operations (add, subtract, multiply, divide) and logical operations (AND, OR, NOT, comparisons).
  • Control Unit (CU): Directs and coordinates the activities of all other units. It fetches instructions from memory, decodes them, and controls their execution.

3. Memory Unit

  • Primary Memory (RAM/ROM): Directly accessible by CPU. RAM is volatile (data lost when power off). ROM is non-volatile (stores BIOS).
  • Secondary Memory: Hard Disk, CD-ROM, Pen Drive — used for permanent storage.

4. Output Unit

Displays or presents processed results to the user. Examples: Monitor, Printer, Speaker, Projector.

Always draw the block diagram. Label Input → CPU (ALU + CU) → Output with Memory connected to CPU. This diagram alone fetches 2 marks.
Q.2 Describe the generation of computers in detail. 6 Marks ▼

Computers have evolved through five generations based on the technology used in their construction:

Generation Period Technology Features Example
1st 1940–1956 Vacuum Tubes Very large, slow, expensive; generated heat; used machine language ENIAC, UNIVAC
2nd 1956–1963 Transistors Smaller, faster, cheaper than 1st gen; used Assembly language IBM 7094
3rd 1964–1971 Integrated Circuits (IC) Multiple transistors on a chip; introduced High-Level Languages (FORTRAN, COBOL) IBM 360
4th 1971–Present Microprocessors (VLSI) Thousands of ICs on one chip; personal computers; GUI; C, C++ used Intel 4004, IBM PC
5th Present–Future AI, ULSI, Parallel Processing Artificial Intelligence, voice recognition, robots; natural language processing IBM Watson

Key Points to Remember

  • Each generation reduced size and cost while increasing speed and capacity.
  • Language evolved: Machine → Assembly → High Level → GUI-based → AI/Natural Language.
  • 4th generation introduced the concept of Personal Computers (PCs).

Q.3 Explain the classification of computers based on size and purpose. 4 / 6 Marks ▼

Classification Based on Size

  • Microcomputers (Personal Computers): Smallest and cheapest. Used by single user. E.g., Desktop, Laptop, Tablet, Smartphone.
  • Minicomputers: Medium-sized, used by multiple users simultaneously. E.g., PDP-11.
  • Mainframe Computers: Large, powerful, used by large organizations (banks, airlines) for bulk processing. E.g., IBM Z-series.
  • Supercomputers: Fastest and most powerful. Used for scientific research, weather forecasting, nuclear simulation. E.g., PARAM (India), Cray.
  • Embedded Computers: Built into devices for specific tasks. E.g., washing machine controller, ATM.

Classification Based on Purpose

  • General-Purpose Computers: Can perform a variety of tasks — word processing, gaming, internet browsing. E.g., Personal computers.
  • Special-Purpose Computers: Designed for one specific task. E.g., ATM machine, Traffic signal controller, Digital camera.

Classification Based on Functionality (Bonus)

  • Analog: Works on continuous signals. E.g., Thermometer, Speedometer.
  • Digital: Works on discrete binary data (0 and 1). E.g., Calculators, PCs.
  • Hybrid: Combination of analog and digital. E.g., Hospital monitors.

Q.5 Discuss memory organization and types of memory in detail. 4 / 6 Marks ▼
MEMORY HIERARCHY ┌─────────────────┐ │ Registers │ ← Fastest, Smallest (inside CPU) ├─────────────────┤ │ Cache Memory │ ← L1, L2, L3 Cache ├─────────────────┤ │ Primary Memory │ ← RAM, ROM │ (Main Memory) │ ├─────────────────┤ │ Secondary Memory│ ← HDD, SSD, CD, Pen Drive └─────────────────┘ ↑ Slower, Larger, Cheaper

Primary Memory (Main Memory)

  • RAM (Random Access Memory): Volatile memory — data is lost when power is off. Stores currently running programs and data. Types: DRAM, SRAM.
  • ROM (Read Only Memory): Non-volatile — data is permanent. Stores BIOS/firmware. Types: PROM, EPROM, EEPROM.
  • Cache Memory: Very fast memory between CPU and RAM. Stores frequently used data to speed up processing.

Secondary Memory (Auxiliary Storage)

Permanent, non-volatile storage. Data is retained even when power is off.

  • Magnetic: Hard Disk Drive (HDD), Floppy Disk
  • Optical: CD, DVD, Blu-Ray
  • Flash: SSD, Pen Drive, Memory Card

Q.6 Explain the types of computer software with suitable examples. 4 Marks ▼

Software is a collection of programs and instructions that tell the computer what to do. It is classified into two main types:

1. System Software

System software manages the hardware and provides a platform for running application software. It acts as an interface between hardware and user.

  • Operating System: Controls all hardware resources. E.g., Windows, Linux, Android, macOS.
  • Compiler/Interpreter: Translates high-level language code to machine code. E.g., GCC for C.
  • Assembler: Converts assembly language to machine language.
  • Device Drivers: Allow OS to communicate with hardware. E.g., Printer driver.
  • Utilities: Perform specific maintenance tasks. E.g., Antivirus, Disk Cleanup.

2. Application Software

Designed to perform specific tasks for the user.

  • General Purpose: MS Word, MS Excel, MS PowerPoint, Web Browsers.
  • Specific Purpose: Tally (accounting), AutoCAD (design), Adobe Photoshop (editing).
  • Mobile Apps: WhatsApp, Google Maps, YouTube.

3. Programming Software (Tools)

Tools that help programmers write code. E.g., IDE (Eclipse, VS Code, Dev-C++), Debuggers, Code Editors.

Q.7 Describe the overview of an Operating System and its functions. 4 Marks ▼

An Operating System (OS) is the most important system software. It acts as an interface between the user and the computer hardware. Without an OS, the computer cannot be used.

Examples: Windows 11, Linux, macOS, Android, iOS.

Functions of an Operating System

  • Process Management: Controls creation, scheduling, and termination of processes/programs.
  • Memory Management: Allocates RAM to programs; ensures no two programs use the same memory space.
  • File Management: Manages files and folders — creation, deletion, reading, writing, and organization.
  • Device Management: Controls input/output devices via device drivers.
  • Security Management: Provides login, passwords, access control to protect data.
  • User Interface: Provides either GUI (Graphical User Interface) like Windows, or CLI (Command Line Interface) like Linux terminal.
  • Error Detection: Detects and reports hardware/software errors.
Q.8 Explain the concept of Networking and types of networks. 4 Marks ▼

A Computer Network is a system where two or more computers are connected together to share resources (data, printers, internet) and communicate with each other.

Types of Networks

  • LAN (Local Area Network): Connects computers within a small area like a building, office, or school. Fast and cheap. E.g., College lab network.
  • MAN (Metropolitan Area Network): Covers a city or large campus. Larger than LAN. E.g., Cable TV networks in a city.
  • WAN (Wide Area Network): Spans a large geographical area — countries or continents. E.g., The Internet.
  • PAN (Personal Area Network): Very small, personal network. E.g., Bluetooth between phone and earphone.

Benefits of Networking

  • Resource sharing (printers, scanners)
  • Data sharing and communication (email, chat)
  • Centralized data storage and backup
  • Cost reduction
Q.9 Describe the process of program development (Editing, Compiling, Debugging). 4 / 6 Marks ▼

Program development is a step-by-step process to create a working software program. The steps are:

PROBLEM ↓ ANALYSIS (Understand the problem) ↓ ALGORITHM (Step-by-step solution in plain English) ↓ FLOWCHART (Pictorial representation of algorithm) ↓ CODING / EDITING (Write code in IDE like Eclipse or Dev-C++) ↓ COMPILATION (Compiler converts code → machine code) ↓ ERROR CHECKING (Fix Syntax/Semantic/Runtime errors) ↓ EXECUTION (Run the program) ↓ TESTING & DEBUGGING (Test with different inputs, fix bugs) ↓ DOCUMENTATION (Write user manual / comments)

Types of Errors

  • Syntax Error: Wrong grammar of programming language. Detected by compiler. E.g., missing semicolon in C.
  • Semantic Error: Code is grammatically correct but logically wrong. E.g., using = instead of == for comparison.
  • Runtime Error: Error that occurs during execution. E.g., division by zero.
  • Logical Error: Program runs but gives wrong output. E.g., using + instead of * in a formula.

IDE (Integrated Development Environment)

An IDE provides a text editor, compiler, debugger, and other tools in one application. Examples: Eclipse (Java, C), Dev-C++, VS Code, Code::Blocks.

Q.10 ⭐ Explain C Tokens in detail with examples. (ASKED IN PREVIOUS PAPER – 4 Marks) 4 Marks ▼
This question was asked in the previous Mid-Sem paper for 4 marks. Write all 6 types of tokens with 1–2 line explanation and example.

A Token is the smallest individual unit in a C program. When a C program is compiled, the compiler reads it character by character and groups characters into tokens. There are 6 types of tokens in C:

1. Keywords

Reserved words with predefined meaning in C. Cannot be used as variable names.

int  float  char  if  else  while  for  return  void

2. Identifiers

User-defined names given to variables, functions, or arrays.

int sum;      // 'sum' is an identifier
float marks;  // 'marks' is an identifier

Rules: Must start with a letter or underscore(_), no special characters, no spaces, not a keyword.

3. Constants

Fixed values that cannot be changed during program execution.

10       // Integer constant
3.14     // Float constant
'A'      // Character constant
"Hello"  // String constant

4. Strings

Sequence of characters enclosed in double quotes, terminated by a null character '\0'.

"Programming"   // String token

5. Operators

Symbols that perform operations on operands.

+   -   *   /   %    // Arithmetic
==  !=  <   >        // Relational
&&  ||  !            // Logical

6. Special Symbols (Punctuators)

Used for specific purposes in syntax.

{ }  // Block delimiters
( )  // Function calls
[ ]  // Array subscripts
;    // Statement terminator
,    // Separator
Write all 6 types clearly with a heading for each and one example. This answer easily scores full 4 marks.

Example Program

#include <stdio.h>

    // Relational operators
    printf("a == b : %d\n", a == b);   // 0

Q.14 Describe Bitwise Operators in C with examples. 4 Marks ▼

Bitwise operators work at the bit level — they perform operations on the binary representation of integers.

OperatorNameOperation
&Bitwise AND1 if both bits are 1
|Bitwise OR1 if either bit is 1
^Bitwise XOR1 if bits are different
~Bitwise NOTInverts all bits (complement)
<<Left ShiftShifts bits left (×2)
>>Right ShiftShifts bits right (÷2)

Example (a=12, b=10)

a = 12 → binary: 1100 b = 10 → binary: 1010 a & b = 1100 & 1010 = 1000 = 8 a | b = 1100 | 1010 = 1110 = 14 a ^ b = 1100 ^ 1010 = 0110 = 6 ~a = ~1100 = ...0011 = -13 (2's complement) a << 1 = 1100 << 1 = 11000 = 24 a >> 1 = 1100 >> 1 = 0110 = 6
#include <stdio.h>
int main() {
    int a = 12, b = 10;
    printf("AND: %d\n", a & b);   // 8
    printf("OR: %d\n", a | b);    // 14
    printf("XOR: %d\n", a ^ b);   // 6
    printf("NOT: %d\n", ~a);      // -13
    return 0;
}
Q.15 ⭐ Explain Conditional (Ternary) Operator with examples. (ASKED IN PREV. PAPER – 6 Marks) 6 Marks ▼
This was asked in Q2b of previous paper (6 marks). Write definition, syntax, explanation, and 2 programs.

The Conditional Operator (also called Ternary Operator) is the only operator in C that takes three operands. It is used as a shorthand for if-else statements.

Syntax

variable = (condition) ? expression_if_true : expression_if_false;

How it Works

  • The condition is evaluated first.
  • If condition is true (non-zero) → the value before : is returned.
  • If condition is false (zero) → the value after : is returned.

Example 1: Find Maximum of Two Numbers

#include <stdio.h>
int main() {
    int a = 10, b = 20, max;
    max = (a > b) ? a : b;
    printf("Maximum = %d\n", max);  // Output: Maximum = 20
    return 0;
}

Example 2: Check Even or Odd

#include <stdio.h>
int main() {
    int n = 7;
    printf("%d is %s\n", n, (n%2==0) ? "Even" : "Odd");
    // Output: 7 is Odd
    return 0;
}

Equivalent if-else

// This ternary: max = (a > b) ? a : b;
// Is same as:
if (a > b)
    max = a;
else
    max = b;

Q.16 Explain Operator Precedence and Order of Evaluation in C. 4 Marks ▼

Operator Precedence determines which operator is evaluated first in an expression that has multiple operators. Operators with higher precedence are evaluated before those with lower precedence.

Associativity determines the direction of evaluation when operators have the same precedence (Left-to-Right or Right-to-Left).

Example

int result = 2 + 3 * 4 - 1;

Q.17 Explain Type Conversion in C (Implicit and Explicit). 4 Marks ▼

Type Conversion is the process of converting a value from one data type to another. There are two types:

1. Implicit Type Conversion (Type Promotion / Automatic)

Done automatically by the compiler when different data types are used in an expression. Lower type is automatically converted to higher type.

int a = 5;
float b = 2.5;
float result = a + b;  // 'a' (int) is auto-converted to float
// result = 7.5

Hierarchy: char → int → long → float → double

2. Explicit Type Conversion (Type Casting / Manual)

Done manually by the programmer using a cast operator to force conversion.

// Syntax: (data_type) expression

int a = 7, b = 2;
float result = (float) a / b;
// Without cast: 7/2 = 3 (integer division)
// With cast: (float)7/2 = 3.5 (float division)
printf("Result = %.1f\n", result);  // Output: 3.5

Q.18 Explain Assignment Operators and Expressions in C. 4 Marks ▼

An Assignment Operator is used to store a value in a variable. The basic assignment operator in C is =.

Simple Assignment

int x = 10;   // assigns 10 to x

Compound Assignment Operators

These combine arithmetic with assignment for shorter syntax:

OperatorExampleEquivalent ToResult (a=10)
+=a += 5a = a + 515
-=a -= 3a = a - 37
*=a *= 2a = a * 220
/=a /= 5a = a / 52
%=a %= 3a = a % 31

Example Program

#include <stdio.h>
int main() {
    int a = 10;
    a += 5;   printf("a += 5 : %d\n", a);   // 15
    a -= 3;   printf("a -= 3 : %d\n", a);   // 12
    a *= 2;   printf("a *= 2 : %d\n", a);   // 24
    a /= 4;   printf("a /= 4 : %d\n", a);   // 6
    a %= 4;   printf("a %%= 4 : %d\n", a);  // 2
    return 0;
}
